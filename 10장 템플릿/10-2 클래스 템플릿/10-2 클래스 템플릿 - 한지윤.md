# 10-2. 클래스 템플릿

# 클래스 템플릿의 개념 및 사용법

클래스 템플릿은 `template` 및 `typename` 키워드를 사용하여 템플릿 매개변수를 선언하고, 이 매개변수를 활용해 **다양한 데이터 형식에 대응할 수 있는 범용 클래스**를 생성하는 방법이다

함수 템플릿과 유사하게, 클래스 템플릿은 특정 데이터 형식에 구애받지 않고 동작하는 클래스를 만들 수 있게 한다

**클래스 템플릿의 선언과 정의:**

```cpp
template <typename Typel, typename Type2>
class data_package {
public:
    data_package(Typel first, Type2 second) : first(first), second(second) {}
private:
    Typel first;
    Type2 second;
};
```

객체를 생성할 때는 `data_package<int, double> template_inst1(5, 10.5);`와 같이 **템플릿 매개변수에 사용할 데이터 형식을 명시적으로 지정**해야 한다

# 클래스 템플릿에서 형식 추론

함수 템플릿과 달리, 전통적으로 클래스 템플릿은 객체 생성 시 템플릿 매개변수의 **형식을 명시적으로 지정**해야 했음

```cpp
// 이전 C++ 표준
data_package<int, double> template_inst1(5, 10.5);
```

하지만 **C++17 표준 이후**부터는 클래스 템플릿에서도 생성자의 인자를 통해 템플릿 매개변수의 형식을 **자동으로 추론**할 수 있게 되어, 형식을 생략하는 것이 가능해졌다

```cpp
// C++17 이후
data_package template_inst1(5, 10.5); // int와 double로 자동 추론
```

그러나 컴파일러가 모든 상황에서 형식을 완벽하게 추론할 수 없으므로, **데이터 형식이 모호할 때는 명시적으로 지정**해 주는 것이 좋음

# 부분 특수화 (Partial Specialization)

부분 특수화는 클래스 템플릿의 템플릿 매개변수 **일부만 특정 형식으로 지정**하여 별도로 처리하는 기능

이는 함수 템플릿에서는 불가능하고 클래스 템플릿에서만 사용할 수 있다

예를 들어, `data_package` 클래스의 첫 번째 템플릿 매개변수를 `string`으로 고정하고 싶을 때 다음과 같이 부분 특수화를 정의할 수 있음

```cpp
template <typename T> // 두 번째 매개변수 T는 여전히 범용
class data_package<string, T> { // 첫 번째 매개변수를 string으로 특수화
public:
    data_package(string first, T second) : first(first), second(second) {}
    void print_out_element() {
        cout << first << "과 함께 입력된, 두 번째 요소: " << second << endl;
    }
private:
    string first;
    T second;
};
```

이 경우, `data_package<string, int> template_inst2("문자열", 10);`와 같이 첫 번째 인자가 `string`인 객체를 생성하면 부분 특수화된 클래스가 인스턴스화된다

# 중첩 클래스 템플릿 (Nested Class Template)

클래스 템플릿 내부에 또 다른 클래스 템플릿을 중첩하여 선언할 수 있다

안쪽 클래스는 바깥쪽 클래스의 템플릿 매개변수를 사용하거나, 새로운 템플릿 매개변수를 정의하여 사용할 수 있다

**중첩 클래스 템플릿의 사용 방법 두가지**

1. **멤버 변수처럼 사용하는 방법**
바깥쪽 클래스에서 안쪽 클래스 템플릿을 멤버 변수로 선언하고 초기화
    
    ```cpp
    template <typename Typel, typename Type2>
    class data_package {
        // ... (생략) ...
    private:
        Typel first;
        Type2 second;
        nested_class_data_package<Type2> internal_data; // 안쪽 클래스를 멤버 변수로 사용
    };
    ```
    
2. **독립된 객체로 선언하여 사용하는 방법**
바깥쪽 클래스의 범위를 지정하여 안쪽 클래스 템플릿 객체를 직접 생성한다
이때 바깥쪽 클래스의 템플릿 매개변수도 함께 명시해야 함
    
    ```cpp
    data_package<string, int>::nested_class<int> template_inst2(500);
    ```
    

중첩된 클래스 템플릿 간에는 상속 관계가 없으므로 접근 지정자는 독립적으로 적용됨

<aside>

🔍 **중첩 클래스 템플릿이란?**
클래스 안에 클래스가 들어가는 것을 "중첩 클래스"라고 하는데, 이게 템플릿으로도 가능하다는 것

- **왜 사용하나?**
    - 논리적으로 서로 밀접하게 관련되어 있지만, 독립적인 기능이나 데이터를 가질 때 사용
    - 예를 들어, `데이터_패키지`라는 큰 개념 안에 `세부_데이터`라는 작은 개념이 있을 때 유용
</aside>

# 템플릿 매개변수 기본값 (Default Template Arguments)

함수 매개변수에 기본값을 설정할 수 있듯이, 템플릿 매개변수에도 **기본 데이터 형식**을 지정할 수 있다
이는 객체 생성 시 템플릿 매개변수의 형식을 생략해도 미리 설정된 기본 형식이 사용되도록 한다

```cpp
template <typename T = int> // T의 기본 형식을 int로 설정
class data_package {
public:
    data_package(T first) : first(first) {}
    // ...
private:
    T first;
};

int main() {
    data_package<> template_inst1(5);         // 기본 형식(int)으로 지정
    data_package<string> template_inst2("hello"); // string 형식으로 지정
    return 0;
}
```

위 예시에서 `data_package<> template_inst1(5);`와 같이 `<>` 안에 아무것도 지정하지 않으면 기본값인 `int`가 사용됨

### 클래스 템플릿 프렌드 (Class Template Friend) (안쓰는게 좋다 함)

클래스 템플릿도 일반 클래스처럼 프렌드 클래스나 프렌드 함수를 가질 수 있다
이때 **인스턴스화 순서**와 **프렌드 규칙**을 준수하는 것이 중요

### 프렌드 클래스 템플릿

프렌드 클래스 템플릿을 선언할 때는 프렌드로 지정될 클래스 템플릿이 **먼저 선언**되어야 합니다.

```cpp
template <typename U>
class caller { /* ... */ }; // caller 클래스 템플릿이 먼저 선언 및 정의

template <typename T = int>
class data_package {
    // ...
    friend caller<data_package>; // data_package<T>의 프렌드로 caller<data_package> 지정
    // ...
};
```

만약 선언 순서가 바뀌면 컴파일 오류가 발생할 수 있다

### 프렌드 함수

프렌드 함수는 클래스 템플릿 내부에 정의하거나 외부에 정의할 수 있음

1. **클래스 템플릿 내부에 정의:** 가장 간단한 방법, 템플릿 매개변수를 그대로 사용할 수 있다
    
    ```cpp
    template <typename T = int>
    class data_package {
    public:
        // ...
        friend void printout_friend_element(data_package<T>& data_object) {
            cout << "(friend 클래스 템플릿 호출) 템플릿 매개변수 값 : " << data_object.first << endl;
        }
    private:
        T first;
    };
    ```
    
2. **클래스 템플릿 외부에 정의:** 가독성을 높이기 위해 클래스 내부에서는 선언만 하고, 정의는 전역 범위에서 할 수 있다
이때 프렌드 함수에서 사용할 **새로운 템플릿 매개변수를 별도로 선언**해야 한다
만약 특정 형식만 수용하는 프렌드 함수를 만들고 싶다면, 정의 시 템플릿 매개변수를 `data_package<C>`와 같이 제한할 수 있다
    
    ```cpp
    template <typename C> // 프렌드 함수를 위한 별도의 템플릿 매개변수 선언
    void printout_friend_element(C &data_object); // 프렌드 함수 선언
    
    template <typename T = int>
    class data_package {
    public:
        // ...
        template <typename C> // 프렌드 함수 선언에도 템플릿 매개변수 필요
        friend void printout_friend_element(C &data_object);
    private:
        T first;
    };
    
    template <typename C> // 프렌드 함수 정의
    void printout_friend_element(C &data_object) {
        cout << "(friend 클래스 템플릿 호출) 템플릿 매개변수 값 : " << data_object.first << endl;
    }
    ```
    
    ```cpp
    template <typename C>
    void printout_friend_element(data_package<C> &data_object) { /* ... */ }
    ```
