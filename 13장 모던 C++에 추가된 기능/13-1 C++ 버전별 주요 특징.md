# 13-1 C++ 버전별 주요 특징

<aside>

- 모던 C++ 언어의 특징 이해하기
- C++98/03과 차별화된 모던 C++ 표준안 이해하기
</aside>

C++98은 언어의 기본 골격을 완성한 최초의 표준이다. 표준에 맞게 작성한 소스 코드는 표준을 지원하는 컴파일러를 사용한다면 종류와 상관없이 컴파일할 수 있다.

C++03은 C++98에서 발견된 문제점을 수정해서 보강한 것이다. 한 가지 큰 특징은 STL이 C++ 표준 라이브러리로 편입되었다는 것이다.

C++11의 큰 특징은 다른 언어에 있는 foreach와 유사한 ‘범위 기반 for 문’을 사용할 수 있게 되었고, 람다 표현식이 도입되었다는 것이다. C++ 표준 위원회는 표준의 품질도 중요하지만 정기적인 표준안 제정이 더 중요하다고 판단하여 C++11부터 3년 단위로 표준을 제정하기로 했다.

C++14는 C++11의 수정 사항들을 반영한 버전이다.

C++17에서는 파일 시스템과 병렬 처리 라이브러리를 도입했으며, 몇 가지 문법 변화도 있었다.

C++20에서는 바이너리 모듈을 가져오는 모듈(module), 템플릿 매개변수에 제약 조건을 명시할 수 있는 컨셉(concept), 그리고 함수를 수시로 멈추었다 다시 진행할 수 있는 코루틴(coroutine) 등 많은 변화가 있었다.

C++23은 코로나19의 영향으로 몇 가지 변경사항이 C++26으로 미루어졌지만, 연역 this 포인터와 컴파일러 최적화 등이 표준으로 채택되었다.

<aside>

# C++11/14 주요 변경 사항

</aside>

C++가 처음 만들어진 1984년과 현재의 컴퓨터 환경은 많이 달라졌다. 이런 변화가 반영되어 C++11/14에는 기존 C++ 언어와는 매우 다른 문법 요소와 표준 라이브러리가 채택되었다. 이는 C++의 장점인 저수준 개발 자원을 유지하면서도 생산성을 높였다.

<br>

## 범위 기반 for 문

인덱스 없이 컨테이너를 순회할 수 있다. Python의 `for x in list`와 유사하다.

```cpp
std::vector<int> v = {1, 2, 3};
for (int x : v) {
    std::cout << x << "\n";
}
```

<br>

## 자료형 추론 auto

컴파일러가 자료형을 추론하므로 긴 타입 이름을 생략할 수 있다.

```cpp
auto x = 42;  // int
auto s = std::string{"hello"};
auto it = v.begin();  // iterator 타입 자동 추론
```

<br>

## 널 포인터 리터럴 nullptr

기존의 `NULL`(매크로, 정수 0)을 대체. 타입 안전하게 null 포인터를 표현한다.

```cpp
void foo(int);
void foo(int*);

foo(NULL);  // 모호함 — int인지 int*인지 불명확
foo(nullptr);  // 명확히 int* 오버로드 호출
```

<br>

## 람다 표현식

람다 표현식(lambda expressions)으로 함수를 값처럼 인라인(inline)으로 작성할 수 있다. 이를 통해 소스 코드가 간결해지고 함수의 흐름을 쉽게 파악할 수 있게 한다.

```cpp
int offset = 10;
auto add = [offset](int x) { return x + offset; };
std::cout << add(5);  // 15
```

<br>

## override와 final 키워드

자식 클래스에서 가상 함수를 선언할 때 override 키워드를 추가하면 부모 클래스의 가상 함수를 오버라이딩했음을 명시할 수 있다. final을 사용하면 더는 오버라이딩하지 않음을 명시할 수 있다. 이렇게 하면 의도와 다르게 코드를 작성했을 때 컴파일 오류를 유도해 적절하게 대처할 수 있다.

```cpp
struct Base { virtual void draw() {} };
struct Child : Base {
    void draw() override {}  // 명시적 오버라이드
    void draw() final {}  // 이후 오버라이드 금지
};
```

<br>

## 스마트 포인터

C++11에서는 메모리를 자동으로 관리해주는 스마트 포인터가 도입되었다. 스마트 포인터는 메모리를 자동으로 해제해 주는 포인터 래퍼이다. `unique_ptr`(단독 소유), `shared_ptr`(공유 소유).

```cpp
auto p = std::make_unique<int>(42);
// 스코프를 벗어나면 자동으로 delete
```

<br>

## 튜플

서로 다른 타입의 여러 값을 하나로 묶어 반환할 수 있다.

```cpp
std::tuple<int, std::string> getInfo() {
    return {42, "hello"};
}
auto [id, name] = getInfo(); // C++17 구조적 바인딩과 함께 사용 가능
```

# C++17 주요 변경 사항

</aside>

## 파일 시스템 라이브러리

OS에 독립적으로 디렉터리·파일을 다룰 수 있는 표준 라이브러리 제공.

```cpp
#include <filesystem>
namespace fs = std::filesystem;
for (auto& entry : fs::directory_iterator("./")) {
    std::cout << entry.path() << "\n";
}
```

<br>

## 병렬 처리 라이브러리

C++17에서는 병렬 처리 방식을 표준화하여 일관된 방식으로 병렬 처리를 할 수 있게 되었다.

<br>

## if와 switch 문 초기화

제어문 안에서 변수를 선언할 수 있어서, 스코프가 해당 블록으로 제한된다.

```cpp
if (auto val = getValue(); val > 0) {
    use(val);
}  // val은 여기서 소멸
```

<br>

## 폴드 표현식

가변 인자 템플릿을 재귀 없이 한 줄로 처리할 수 있다. 메모리 문제나 종료 조건 설정에 대한 실수를 방지할 수 있다.

```cpp
template<typename... Args>
auto sum(Args... args) { return (args + ...); }
sum(1, 2, 3, 4); // 10
```

<br>

## 구조적 바인딩

구조적 바인딩은(structured bindings)은 튜플·구조체의 멤버를 한번에 개별 변수로 꺼낼 수 있다.

```cpp
auto [x, y, z] = std::make_tuple(1, 2.0, "hi");
auto [key, val] = *myMap.begin();
```

# C++20 주요 변경 사항

</aside>

## 컨셉

템플릿 매개변수에 제약 조건을 명시해, 잘못된 타입 사용 시 명확한 오류 메시지를 제공한다.

```cpp
template<typename T>
concept Numeric = std::is_arithmetic_v<T>;

template<Numeric T>
T add(T a, T b) { return a + b; }
// add("a", "b") → 컴파일 오류 (명확한 메시지)
```

<br>

## 코루틴

코루틴(coroutines)은 함수를 멈췄다가 다시 호출할 수 있는 루틴을 말한다. 함수를 중간에 멈추고 재개할 수 있다. 지역 변수가 중단 사이에도 유지된다.

```cpp
Generator counter() {
    int n = 0;
    while (true) co_yield n++;  // 여기서 멈추고 값 반환
}
auto gen = counter();
gen.next();  // 0
gen.next();  // 1
```

<br>

## 모듈

`#include` 대신 미리 컴파일된 모듈을 가져올 수 있어 빌드 속도가 빠르고 선언 중복을 신경 쓸 필요가 없다.

```cpp
// 기존
#include <iostream>

// 모듈
import std.core;
```

<br>

## 3방향 비교 연산자

3방향 비교 연산자(three-way comparison)는 작음(<), 같음(=), 큼(>) 등 3가지 방향을 조합해 총 5가지 비교를 한번에 할 수 있다. 우주선 연산자라고도 부른다.

```cpp
auto result = 3 <=> 5;
// result < 0  → 3이 작음
// result == 0 → 같음
// result > 0  → 3이 큼
```

<br>

## 수학 상수

π, e, √2 등 자주 쓰이는 수학 상수가 표준 라이브러리에 추가되었다.

```cpp
#include <numbers>
double area = std::numbers::pi * r * r;
```

## 연역 this 포인터

멤버 함수의 `this`를 명시적 매개변수로 받아, `const`·`ref` 등의 수식어를 유연하게 적용할 수 있다.

```cpp
struct Widget {
    void show(this const Widget& self) {
        // self는 읽기 전용 this
    }
};
```

<br>

## 모나드 형식 optional, expected

모나드(monad)는 값의 존재 여부나 오류를 체이닝 방식으로 처리할 수 있다.

```cpp
std::optional<int> parse(std::string s) { ... }

parse("42")
  .and_then([](int x) { return x * 2; })  // 값이 있을 때
  .or_else([]() { return 0; });           // 없을 때
```

<br>

## 가변 인자 출력 print

C++23 표준 라이브러리에 추가된 print 함수는 `cout`의 장황함과 `printf`의 타입 불안전성을 모두 해소한 출력 함수이다.

```cpp
#include <print>
std::print("이름: {}, 나이: {}\n", name, age);
// printf보다 안전, cout보다 간결
```
