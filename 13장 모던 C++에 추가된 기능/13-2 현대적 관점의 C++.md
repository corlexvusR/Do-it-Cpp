## char\* 대신 string, []array 대신 vector<> 사용하기

`char*`는 버퍼 크기를 직접 관리해야 해서 오버플로우 위험이 있다. `string`은 크기를 자동으로 관리하고 다양한 변환 메서드를 제공한다. 마찬가지로 `vector`는 크기가 동적으로 늘어나며 범위 기반 `for`문도 사용할 수 있다.

<br>

**C 스타일**

```c
char name[20];
strcpy(name, "hello");
strcat(name, " world");

int arr[5] = { 1, 2, 3, 4, 5 };
int n = sizeof(arr) / sizeof(arr[0]);
```

<br>

**모던 C++**

```cpp
std::string name = "hello";
name += " world";  // 크기 자동 조정

std::vector<int> arr = { 1, 2, 3, 4, 5 };
for (int x : arr) { ... }
```

## 포인터 대신 레퍼런스 사용하기

포인터는 `*`, `->`, `&` 연산자를 혼용해야 해 읽기 어렵다. 레퍼런스는 선언 시 `&` 한 번만 쓰면 되고, 이후 사용은 일반 변수와 동일하다. 단, null이 될 수 없는 참조가 필요할 때 적합하다.

<br>

**포인터**

```c
void increment(int* p) {
    if (p) (*p)++;
}
int x = 0;
increment(&x);
```

<br>

**레퍼런스**

```cpp
void increment(int& n) {
    n++;
}
int x = 0;
increment(x);
```

<br>

## 메모리 직접 관리 대신 RAII 사용하기

RAII(Resource Acquisition Is Initialization)는 자원의 획득과 해제를 객체의 생명주기에 묶는 패턴이다. `new`/`delete` 쌍을 직접 맞추는 대신, 스마트 포인터나 컨테이너처럼 소멸자에서 자동으로 해제되는 객체를 사용한다. 스코프를 벗어나면 예외가 발생하더라도 자원이 반드시 해제된다.

<br>

**수동 관리**

```c
int* p = new int(42);
// ... 예외 발생 시 누수
delete p;
```

<br>

**RAII**

```cpp
auto p = std::make_unique<int>(42);
// 스코프 종료 시 자동 해제
// 예외가 발생해도 안전
```
