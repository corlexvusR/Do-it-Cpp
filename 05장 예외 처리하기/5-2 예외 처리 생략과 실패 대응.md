# 예외 처리 생략 - noexcept

</aside>

함수에서 문제가 발생할 때마다 예외를 던져야 하는 것은 아니며, 예외를 발생시키는 것이 득보다 실이 더 클 때가 있다. 오류와 예외는 다르므로, 값이나 실행 흐름을 예측할 수 있을 때에는 if 문으로 처리하는 것이 성능 면에서 낫다.

<br>

함수가 예외를 던지지 않음을 나타낼 때는 noexcept 키워드로 명시할 수 있다. 이런 명시는 컴파일러가 코드를 최적화하고 빠르게 실행하는 데 도움이 된다.

함수가 예외를 던지지 않음을 명시

`int func() noexcept`

<br>

또는 다음처럼 함수를 호출할 때 noexecpt 키워드를 사용할 수도 있다. 그러면 컴파일할 때 해당 함수가 예외를 던지는지 확인해서 true나 false로 알려준다.

함수가 예외를 던지는지 확인

`bool does_not_throw = noexcept(my_function());`

noexcept 키워드를 붙였다고 해서 예외를 던지지 못하는 것은 아니다. 다음을 통해 noexcept가 명시된 함수에서 예외를 던지면 어떻게 되는지 볼 수 있다.

<br>

### noexcept로 명시된 함수에서 예외 던지기

```cpp
#include <iostream>
using namespace std;

void real_noexcept() noexcept
{
	cout << "real_noexcept" << endl;
}

// noexcept로 명시된 함수 안에서 예외 발생
void fake_noexcept() noexcept
{
	cout << "fake_noexcept" << endl;
	throw 1; // 정수 형식 예외 발생
}

int main()
{
	real_noexcept();

	try
	{
		fake_noexcept();
	}
	catch (int exec)
	{
		cout << "catch " << exec << endl;
	}

  return 0;
}
```

<br>

**컴파일 결과 (경고 발생)**

`warning C4297: ‘fake_noexcept’: 함수는 예외를 Throw하지 않도록 지정되었으나 예외를 Throw했습니다.`

`message: 함수에서 __declspec(nothrow), throw(), noexcept(true) 또는 noexcept를 지정했습니다.`

이 코드를 컴파일하면 경고가 발생하지만 실행 파일은 만들어진다. 컴파일러는 noexcept 키워드가 붙은 함수는 예외를 던지지 않을 것이라 간주하고 컴파일을 진행한다. 그러나 실행 중에 런타임 오류 메시지가 발생하고 프로그램은 강제 종료된다.

<br>

# 예외 처리 실패 대응 - set_terminate

</aside>

throw 문으로 예외를 던졌지만 해당 예외를 받을 catch 문이 없거나 있어도 형식이 맞지 않으면 프로그램은 강제 종료된다. 이때 종료 처리 함수를 설정하는 set_terminate로 프로그램이 강제 종료되기 전에 특정 동작을 수행하도록 설정할 수 있다.

종료 처리 함수 설정하기

`set_terminate(종료_처리_함수);`

<br>

## 실습: 예외 처리 실패에 대응하기

```cpp
#include <iostream>
#include <cstdlib>  // exit와 set_terminate 함수 사용을 위해 추가

using namespace std;

// 종료 처리 함수
void myterminate() {
  cout << "myterminate called" << endl;
  exit(-1);      // 프로그램을 비정상적으로 종료
}

int main(void) {
  set_terminate(myterminate);    // 종료 처리 함수를 myterminate로 지정

  throw 1;    // 처리되지 않는 예외를 던짐

  return 0;   // 이 줄은 실행되지 않음, 위에서 예외가 던져졌기 때문
}
```

실험 결과: `myterminate called`

main 함수에서 throw 1 코드를 만나면 catch 문을 찾을 수 없어 오류가 발생하고 프로그램이 강제로 종료된다. 강제로 종료되기 전에 set_terminate로 지정한 종료 처리 함수가 호출된다.

<br>

### exit 함수

프로그램을 강제로 종료하는 함수이며, exit 함수는 <cstdlib> 헤더에 정의되어 있다. exit 함수에 상태 코드로 0을 전달하면 프로그램을 정상으로, -1을 전달하면 비정상으로 종료한다. 이 상태 코드에 따라 내부에 정의된 각종 정리 작업을 수행한다.

비슷한 역할은 하는 abort 함수가 있으며, 이 함수는 정리 작업을 수행하지 않고 프로그램을 비정상으로 즉시 종료한다.
