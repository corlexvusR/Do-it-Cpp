# 5-1. 예외 처리 구문

<aside>

- try, catch, throw 문으로 예외 처리하기
- assert로 예외 처리하기
</aside>

소스 코드에 예외 처리를 하면 예상치 못한 상황에서 유연하게 대처할 수 있다. 이는 코드의 안정성을 높이고 오류로 인한 비정상 종료를 막을 수 있게 한다. 또한 가독성을 높이고 유지 보수와 디버깅을 쉽게 한다.

C++ 언어에서는 예외가 발생했음을 명시적으로 구분하고 예외를 간단하게 처리할 수 있도록 try, catch, throw 문을 제공한다.

<br>

<aside>

# try, catch, throw 문으로 예외 처리

</aside>

C++에서 제공하는 `try`, `catch`, 그리고 `throw`는 예외가 발생했을 때 프로그램의 제어 흐름을 변경하고 예외를 처리하는데 사용된다.

- `try`: 예외가 발생할 수 있는 코드 블록을 중괄호 `{}`로 감싸서 사용한다.
- `throw`: 예외를 catch 블록으로 던진다.
- `catch`: `throw`로 던진 예외를 받아서 처리한다.

<br>

```cpp
try
{
	// 예외를 던질 수 있는 코드
	throw 예외 값
}
catch (예외_형식 예외_이름)
{
	// 예외를 처리하는 코드
}
```

<br>

## 실습: 예외 처리

```cpp
#include <iostream>

using namespace std;

int main() {
  try {
    int input;
    cout << "정수 중 하나를 입력하세요 : ";
    cin >> input;

    // 입력받은 숫자가 양수이면
    if (input > 0) {
      cout << "throw 1" << endl;
      throw 1;      // 예외 1 발생(정수 형식 예외)
      cout << "after throw 1" << endl;
    }

    // 입력받은 숫자가 음수이면
    if (input < 0) {
      cout << "throw -1.0f" << endl;
      throw - 1.0f;   // 예외 1.0f 발생(부동소수점 형식 예외)
      cout << "after throw -1.0f" << endl;
    }

    // 입력받은 숫자가 0이면
    if (input == 0) {
      cout << "throw Z" << endl;
      throw 'Z';    // 예외 Z 발생(문자 형식 예외)
      cout << "after throw Z" << endl;
    }
  }
  catch (int a) {   // 정수 형식 예외 받기
    cout << "catch " << a << endl;
  }
  catch (float b) {   // 부동소수점 형식 예외 받기
    cout << "catch " << b << endl;
  }
  catch (char c) {   // 문자 형식 예외 받기
    cout << "catch " << c << endl;
  }

  return 0;
}
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력하세요 : 1
throw 1
catch 1
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력하세요 : -1
throw -1.0f
catch -1
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력하세요 : 0
throw Z
catch Z
```

<br>

### 예외 발생 시 흐름

```cpp
try
{
	if (...) throw 1;  // 정수 형식 예외 발생
	if (...) throw -1.0f;  // 부동 소수점 형식 예외 발생
	if (...) throw 'Z';  // 문자 형식 예외 발생
}
catch (int a)
{
}
catch (float b)
{
}
catch (char c)
{
}
```

# Catch(…) 문으로 기타 예외 처리

</aside>

## 예외는 던졌지만 받아주는 catch 문이 없다면?

```cpp
#include <iostream>
using namespace std;

int main()
{
	try
	{
		throw 1;  // 예외 1발생 (정수 형식의 예외)
	}
	catch (char c)
	{
		cout << "catch " << c << endl;
	}

	return 0;
}
```

정수 형식 예외를 throw로 던졌지만, 문자 형식 예외를 받는 catch만 있는 상태이다. 실행하면 런타임 오류가 발생하며 프로그램이 강제로 종료된다. 예외를 던졌다면 이를 받는 구문이 있어야 하기 때문이다.

하지만 모든 예외의 짝을 맞춰서 예외 처리 코드를 작성하는 것은 쉽지 않지만, C++ 언어의 예외 처리에서는 catch 문으로 이를 처리할 수 있다. catch 문은 명시하지 않은 나머지 모든 예외를 받아서 처리할 때 사용할 수 있다.

<br>

## 실습: 처리되지 않은 모든 예외 받기

```cpp
#include <iostream>

using namespace std;

int main() {
  try {
    int input;
    cout << "정수 중 하나를 입력해보세요 : ";
    cin >> input;

    // 입력받은 숫자가 양수이면
    if (input > 0) {
      cout << "throw 1" << endl;
      throw 1;   // 예외 1 발생(정수 형식 예외)
    }

    // 입력받은 숫자가 음수이면
    if (input < 0) {
      cout << "throw -1.0f" << endl;
      throw - 1.0f;   // 예외 1.0f 발생(부동소수점 형식 예외)
    }

    // 입력받은 숫자가 0이면
    if (input == 0) {
      cout << "throw Z" << endl;
      throw 'Z';   // 예외 Z 발생 (문자 형식 예외)
    }
  }
  catch (int a) {   // 정수 형식 예외 받기
    cout << "catch " << a << endl;
  }
  catch (...) {   // 처리되지 않은 나머지 예외 모두 받기
    cout << "catch all" << endl;
  }

  return 0;
}
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력해보세요 : 1
throw 1
catch 1
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력해보세요 : -1
throw -1.0f
catch all
```

<br>

### 실행 결과

```cpp
정수 중 하나를 입력해보세요 : 0
throw Z
catch all
```

결과를 보면 정수 형식을 제외한 나머지 예외는 모두 `catch all`이라는 메시지를 출력하는 것을 확인할 수 있다.

**그러나 예외는 종류마다 제대로 구분해서 처리하는 편이 더 적절한 예외 처리라고 할 수 있다 .**

# 예외가 전달되는 순서

</aside>

try, catch, throw와 같은 구문들은 예외가 발생했을 때 프로그램의 제어 흐름을 변경하고 오류를 처리하는 데 사용된다. 다음 코드처럼 예외를 던진 func_throw 함수에 catch 문이 없는 상황을 가정해보자.

<br>

## 실습: 예외가 전달되는 동작 확인하기

```cpp
#include <iostream>

using namespace std;

void func_throw() {
  cout << "func_throw()" << endl;
  cout << "throw -1" << endl;
  throw -1;   // 정수 형식 예외 던지기
  cout << "after throw -1" << endl;
}

int main() {
  try {
    func_throw();
  }
  catch (int exec) {   // 정수 형식 예외 받기
    cout << "catch " << exec << endl;
  }

  return 0;
}
```

<br>

### 실행 결과

```cpp
func_throw()
throw -1
catch -1
```

실행 결과를 보면 func_throw 함수에서 던진 예외가 이 함수를 호출한 main 영역의 catch 문에서 정상적으로 처리된 것을 확인할 수 있다. 그러므로 예외 처리의 책임은 throw가 발생한 함수를 호출한 쪽으로 넘어감을 알 수 있다. 이와 같이 함수에서 예외가 처리되지 않아서 함수를 호출한 쪽으로 전달되는 현상을 **스택 풀기**(stack unwinding)라고 한다.

예외가 발생하면 해당 함수의 남은 코드들은 더 이상 수행되지 않고 종료되므로, func_throw 함수에서 ‘after throw -1’은 출력되지 않는다.

<br>

## 실습: 스택 풀기 동작 확인하기

```cpp
#include <iostream>

using namespace std;

void func_throw() {
  cout << endl;
  cout << "func_throw() 함수 내부" << endl;
  cout << "throw -1" << endl;
  throw - 1;   // 정수 형식 예외 던지기
  cout << "after throw -1" << endl;
}

void func_2() {
  cout << endl;
  cout << "func_2() 함수 내부" << endl;
  cout << "func_throw() 호출" << endl;
  func_throw();
  cout << "after func_throw()" << endl;
}

void func_1() {
  cout << endl;
  cout << "func_1() 함수 내부" << endl;
  cout << "func_2() 호출" << endl;
  func_2();
  cout << "after func_2()" << endl;
}

int main() {
  cout << "main 내부" << endl;

  try {
    cout << "func_1() 호출" << endl;
    func_1();
  }
  catch (int exec) {   // 정수 형식 예외 받기
    cout << endl;
    cout << "catch " << exec << endl;
  }

  return 0;
}
```

<br>

### 실행 결과

```cpp
main 내부
func_1() 호출

func_1() 함수 내부
func_2() 호출

func_2() 함수 내부
func_throw() 호출

func_throw() 함수 내부
throw -1

catch -1
```

<br>

이 예에서 함수가 호출되는 순서는 다음과 같다.

```cpp
main -> func_1() -> func_2() -> func_throw()
```

함수가 호출되면 스택에 순서대로 쌓이게 되므로, 맨 마지막에 호출되는 함수가 가장 위쪽에 위치하게 된다.

예외를 전달한 함수는 남은 코드를 실행하지 않고 바로 종료된다. 그러므로 각 함수에서 마지막 코드인 ‘after~~’를 출력하는 구문은 모두 수행되지 않는다.

이처럼 예외를 전달하는 순서가 스택에 쌓인 역순이므로 **스택 풀기**라고 한다.

# 어설션을 이용한 예외처리

</aside>

어설션(assertion)은 코드를 검증하여 예상치 못한 상황에서 프로그램 동작을 중단시키는 도구이며, 신뢰성과 안정성을 높인다. C++에서는 <cassert> 헤더에 정의된 assert 매크로를 통해 오류가 생길 수 있는 부분을 검사하고 예외를 간단하게 처리할 수 있다.

<br>

## 실습: assert를 이용한 예외 처리

```cpp
#include <iostream>
#include <cassert>

using namespace std;

void print_number(int* _pt_int)
{
  assert(_pt_int != NULL);  // <- 디버그 모드에서 _pt_int가 널인지 검사
  cout << *_pt_int << endl;
}

int main() {
  int a = 100;
  int* b = NULL;
  int* c = NULL;

  b = &a;
  print_number(b);

  // c는 NULL인 상태로 인자 전달
  print_number(c);

  return 0;
}
```

assert를 이용해 프로그램의 특정 지점에서 true로 예상되는 조건을 지정한다. 이때, 지정한 조건이 true가 아니면 프로그램이 실행 중단된다.

assert는 주로 개발 과정에서 조건을 검사하여 프로그램이 예상대로 동작하는지 확인하는 용도로 사용된다.

<br>

## 디버그 모드와 릴리즈 모드

디버그(debug) 모드는 컴파일할 때 디버깅을 위한 정보를 삽입해서 문제가 발생할 때 원인을 수월하게 찾을 수 있게 하는 컴파일 모드이다.

릴리즈(release) 모드는 최종 사용자에게 배포할 코드를 만들 때 사용한다. 릴리즈 모드에서는 최적화가 적용되어 프로그램의 실행 속도가 향상된다.

개발할 때는 주로 디버그 모드를 사용하여 버그를 빠르게 찾아내고, 배포할 때는 릴리즈 모드로 컴파일하여 최적화된 실행 속도와 작은 실행 파일을 만든다.

<br>

## assert를 사용할 때 주의할 점

assert는 디버그 모드에서만 컴파일 되므로 다른 코드에 영향을 주지 않는 코드로만 작성해야 한다.

<br>

### 어선셜을 잘못 사용한 예

```cpp
assert (count++ > 0);
```

assert 검사는 릴리즈 모드에서 수행되지 않으므로 count값이 디버그 모드와 릴리즈 모드에서 다르다. 프로그램의 중요한 코드를 assert 구문에 작성하면, 릴리즈 모드에서는 해당 코드가 빠지게 되므로 유의해야 한다.
